# Stage 5：AI 测试助手 / LLM 对话中心

## 本阶段目标

参考 WHartTest 的 LLM 对话中心，但不做通用聊天机器人，而是做面向测试开发的 AI 工作台：提示词模板、知识库上下文、测试用例生成、失败分析、Bug 报告草稿和可测性分析。

## 已完成能力

| 能力 | 说明 |
| --- | --- |
| `GET /api/assistant/templates` | 返回内置测试开发提示词模板 |
| `POST /api/assistant/chat` | 根据模板、输入、知识库开关生成测试产物 |
| Vue AI 测试助手页面 | 支持模板选择、知识库开关、source_type 过滤、上下文展示 |
| 智能用例生成 | 生成结构化测试用例和 Markdown 草稿 |
| 接口测试设计 | 生成接口测试维度、步骤、断言和 pytest 建议 |
| 失败原因分析 | 根据失败日志生成根因分类、证据和修复建议 |
| Bug 报告生成 | 生成可复制到缺陷平台的结构化 Bug 草稿 |
| 可测性分析 | 识别需求缺失的验收标准、边界、异常和日志要求 |

## 设计说明

当前阶段默认使用 deterministic / rule-based 生成，不强依赖外部 LLM key，保证面试现场可以稳定演示。页面中的“知识库上下文”使用本地测试知识片段模拟 RAG 召回形态，后续 Stage 6 会接入真实知识库上传、检索和引用来源。

## 演示建议

1. 打开 `AI 测试助手` 页面。
2. 选择 `智能用例生成`。
3. 输入：`登录接口支持账号密码登录，成功返回 token，错误密码返回 401，请生成接口测试用例。`
4. 勾选 `api_doc`、`standard`、`bug`。
5. 点击 `生成测试产物`。
6. 说明：平台通过测试模板和 RAG 上下文生成用例，再进入 API 测试和自动化执行。

## 下一阶段

Stage 6 应该做 `知识库与 RAG 页面`：文档上传、知识库列表、检索测试、召回片段、source_type / module / version 过滤和引用来源展示。
